# Spec v0.1

## 1. 問題定義
多人長對話中，洞見容易停留在對話流，無法穩定傳承到下一輪決策。
本專案要把「短期對話記憶」轉為「可審閱資產」。

## 2. 輸入規格
- 檔案格式：Markdown
- 最低結構：
  - 主題標題
  - 至少 2 位參與者發言
  - 每段發言含 `From:` 與時間

## 3. 輸出規格
輸出檔名：`<topic>__consensus-note.md`

輸出段落固定包含：
1. 已收斂共識
2. 保留分歧
3. 待驗證命題
4. 最小可執行實驗（可選，但建議）
5. 目前最大風險
6. 建議主控決策

## 4. 抽取規則（初版）
- 共識：至少兩位參與者明確接受，或無異議延續。
- 分歧：存在明確對立立場，且尚未被實驗或決策消解。
- 待驗證命題：包含可測試語句但缺乏結果。
- 風險：含「最擔心的風險」或等價風險敘述。

## 5. 決策門檻
若輸入不滿足最低結構，工具應回報：
- 缺失欄位
- 建議補件
- 不輸出偽摘要

## 6. 驗收標準
1. 結構驗收：輸出一定包含 1~6 段。
2. 可讀驗收：每段不超過 7 行重點條列。
3. 穩定驗收：相同輸入三次輸出段落順序一致。
4. 邊界驗收：對缺失結構輸入不產生誤導摘要。

## 7. 已知風險
1. 「策略性迎合」等概念仍有標註主觀性。
2. 多人語氣差異可能干擾共識抽取。
3. 極長對話需先分段摘要再整合。
